# Поддержка объектов классов в `describe_call`

## Кратко
`describe_call` будет принимать первым аргументом строковое имя класса или сам объект класса. Для объекта класса определение будет идти через `inspect.isclass` (`isclass`), затем берется `.__name__`, и дальше используется тот же путь проверки имени, лимитов и форматирования, что и для строк.

## Ключевые изменения
- Первым шагом реализации сохранить этот план в `docs/plans/2.md`.
- Обновить `AGENTS.md`, добавив правила:
  - работать строго по TDD: сначала тесты, затем проверка их “красноты”, потом реализация;
  - новые и изменяемые `pytest.raises` с проверкой текста исключения писать только как `match=match(...)`;
  - планы тестов разделять на runtime-тесты и typing-тесты;
  - каждый новый или измененный тест должен иметь англоязычную, план-независимую и лаконичную докстрингу.
- В `printo/describe.py` переименовать первый параметр публичной сигнатуры с `class_name` на `class_or_name` и аннотировать как `Union[str, Type[Any]]` для Python `>=3.8`.
- Внутри `describe_call` инлайнить нормализацию без новых helper-функций: `str` использовать как есть, `isclass(class_or_name)` заменить на `class_or_name.__name__`, все остальное отклонять через `TypeError`.
- Добавить `from inspect import isclass` и `Type` из `typing`; `@repred` не менять.
- Обновить связанные сообщения исключений: `TypeError`, invalid identifier `ValueError`, keyword `ValueError`, `total_limit` too small.
- Обновить README только в связанных местах: описание первого аргумента и формулу минимума `total_limit` как `len(resolved class name) + 2`.

## Проверка
- После написания тестов и до реализации запустить релевантные тесты и убедиться, что новые проверки падают по ожидаемой причине.
- После реализации запустить:
  - `pytest tests/units/test_describe.py tests/typing/test_describe_call.py`
  - `mypy printo --strict`
  - `mypy tests`
  - `coverage run --source=printo --omit="*tests*" -m pytest --cache-clear --assert=plain && coverage report -m --fail-under=100`

## Допущения
- Определение класса выполняется только через `inspect.isclass`, не через `isinstance(value, type)`.
- Специальные тесты на сломанный или нестроковый `.__name__` не добавляются.
- Новые helper-функции не создаются; изменение минимально и остается внутри `describe_call`.
- Старое keyword-имя `class_name=` не сохраняется, так как параметр публично переименовывается в `class_or_name`.
